# 跨功能视觉设计决策

## 一致的视觉语言

### 统一的元数据结构

每个文档文件头部使用相同的元数据模板：

```markdown
---
title: [功能/模块名称]
status: draft | review | final
last_updated: 2026-03-30
dependencies: [依赖的其他文档]
---
```

**视觉一致性原则**：
- 元数据字段顺序固定
- status 值限定在：draft → review → final
- 日期格式统一为：YYYY-MM-DD

### 统一的章节结构

每个教程文档采用相同的章节模板：

```markdown
## 概述

[1-2 段落说明]

## 核心概念

[关键概念列表 + 简短定义]

## 实现步骤

### 步骤 1：[标题]

[说明 + 代码示例]

### 步骤 2：[标题]

[说明 + 代码示例]

## 常见问题

### 问题 1
**现象**：[描述]
**原因**：[分析]
**解决**：[方案]

## 参考资源

- [相关文档链接]
- [API 参考]
```

### 代码块内注释规范

**TypeScript/JavaScript**
```typescript
/**
 * 函数目的：简短描述
 * @param param1 - 参数说明
 * @returns 返回值说明
 */
function example(param1: string): boolean {
  // 实现逻辑
  // 注意：[特殊说明]
  return true;
}
```

**Bash 命令**
```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 注意：端口 3000 必须未被占用
```

### 文件路径展示

**内联路径**：使用反引号
```
配置文件位于 `src/config/task.json`
```

**代码块标题**：标注完整路径
````markdown
```typescript title="src/core/task-context.ts"
````
```

**路径分隔符**：始终使用正斜杠 `/`（跨平台兼容）

## 响应式考虑

### 终端兼容性

**宽度限制**
- ASCII 图表：最大 76 字符
- 代码行：建议 ≤ 80 字符，必要时 ≤ 100
- 表格：确保在 80 列终端可读

**字符编码**
- 使用 UTF-8 编码
- 框线字符：`│ ─ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼`
- 避免使用复杂 Unicode 符号（可能显示异常）

### 移动端适配

虽然 Markdown 主要在桌面端阅读，但考虑移动端：

```markdown
# 避免：过长的代码行
const veryLongVariableName = someFunction(with, many, arguments, that, exceed, screen, width);

# 推荐：适当换行
const result = someFunction(
  with,
  many,
  arguments
);
```

## 视觉决策记录

### 决策 1：使用 ASCII 而非图片

**理由**：
- 纯文本，无外部依赖
- 可版本控制，易于修改
- 支持所有 Markdown 渲染器

**权衡**：
- 优点：轻量、灵活
- 缺点：复杂图表绘制耗时

### 决策 2：统一使用 2 空格缩进

**理由**：
- 与 Markdown 源码一致
- 在列表嵌套时对齐自然

**示例**：
```markdown
- 一级列表
  - 二级列表
    - 三级列表
```

### 决策 3：代码块始终标注语言

**理由**：
- 启用语法高亮
- 明确代码类型
- 支持渲染器优化

**格式**：
````markdown
```language [optional-title]
code here
```
````

## 设计模式总结

### 元模式：渐进式披露

```
概述 → 核心概念 → 基础示例 → 高级用法 → 边缘情况
```

每个章节遵循：
1. **是什么**（概念定义）
2. **为什么**（设计动机）
3. **怎么做**（代码示例）
4. **注意点**（最佳实践）

### 视觉锚点模式

**每 300-500 字设置视觉锚点**：
- 代码块（最有效的锚点）
- ASCII 图表（视觉间隔）
- 引用块（强调重点）
- 列表（信息分解）

---

**设计师注**：跨功能一致性不是限制创意，而是建立可预测的阅读体验。当读者熟悉文档结构后，认知负荷降低，学习效率提升。
