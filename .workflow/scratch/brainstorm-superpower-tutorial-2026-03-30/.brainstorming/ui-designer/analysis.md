# UI 设计师视角分析

## 视觉设计原则

### 清晰层次 (Visual Hierarchy)

SuperPower 教程采用纯 Markdown 格式，视觉层次完全依赖文本结构：

**标题体系**
- `#` 主标题：章节标题，每个文档仅一次
- `##` 次标题：主要概念分组
- `###` 三级标题：具体概念或步骤
- `####` 四级标题：细节说明（慎用，避免过度嵌套）

**视觉强调策略**
```
- **加粗文本**：关键概念、术语首次出现
- `代码样式`：技术术语、文件路径、命令
- > 引用块：重要提示、警告、最佳实践
- --- 分隔线：章节之间的视觉暂停
```

### 一致性原则

**术语一致性**
- 技术术语首次出现时使用 `代码样式` + 中文解释
- 后续引用保持一致的大小写和格式
- 例如：`SuperPower`（始终大写 S 和 P）、`TaskContext`

**代码块一致性**
- 所有代码块标注语言：```typescript、```bash、```markdown
- 文件路径作为标题：```typescript title="src/core/task.ts"
- 长代码块添加注释：`// ... 完整实现见 src/...`

### 可扫描性 (Scannability)

**信息密度控制**
- 每段不超过 4 行
- 每个列表不超过 7 项（认知负荷阈值）
- 代码示例前后的解释文本不超过 50 字

**视觉锚点**
```
## 核心概念

### 概念 A
[简短定义]

**使用场景**：[1-2 句话]

**代码示例**：
[代码块]

> 💡 **提示**：[补充说明]
```

## 文档可读性策略

### 纯文本约束下的视觉设计

**无外部依赖**
- 不使用图片（ASCII 图表替代）
- 不依赖外部 CSS 或样式库
- 所有视觉效果通过 Markdown 原生语法实现

**ASCII 图表设计规范**
```
最大宽度：76 字符（兼容终端显示）
对齐方式：左对齐 + 2 空格缩进
边框字符：
  - 单线：│ ─ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼
  - 双线：║ ═ ╔ ╗ ╚ ╝ ╠ ╣ ╦ ╩ ╬
```

### 代码展示规范

**语法高亮要求**
```markdown
```typescript
// TypeScript 代码：类型标注、接口定义
```javascript
// JavaScript 代码：ES6+ 语法
```bash
# Shell 命令：注释用 #，命令直接写
```markdown
# Markdown 元素：展示 Markdown 语法
```

**代码块最佳实践**
1. **完整性**：示例代码可独立运行
2. **注释比例**：注释占 20-30%
3. **错误处理**：展示 try-catch 或错误处理逻辑
4. **类型安全**：TypeScript 代码必须有类型定义

## 设计检查清单

### 发布前验证

- [ ] 所有代码块有语言标注
- [ ] ASCII 图表宽度 ≤ 76 字符
- [ ] 术语首次出现有解释
- [ ] 每个章节有明确的视觉层次
- [ ] 代码示例可独立运行
- [ ] 无硬编码路径（使用相对路径或占位符）
- [ ] 引用块用于重要提示
- [ ] 列表不超过 7 项

### 可访问性考虑

- [ ] 对比度足够（纯文本默认符合）
- [ ] 表格有表头
- [ ] 链接文本有意义（避免"点击这里"）
- [ ] 代码注释解释"为什么"而非"是什么"

## 视觉风格指南

### 色彩使用（纯文本环境）

虽然 Markdown 不支持颜色，但可以通过符号传达视觉层次：

```
📦 模块/组件
🔧 工具/函数
⚠️  警告
💡 提示
✅ 成功路径
❌ 错误路径
```

### 间距规范

```markdown
# 章节标题

[空行]

段落内容。

[空行]

## 次级标题

[空行]

- 列表项 1
- 列表项 2

[空行]

```代码```

[空行]

后续内容。
```

---

**设计师注**：纯文本约束不是限制，而是回归内容本质。优秀的文档通过结构、层次、一致性传达信息，而非依赖视觉装饰。
