### 代码注释策略

**注释密度规则**：

| 代码复杂度 | 注释比例 | 注释类型 |
|-----------|---------|---------|
| 简单示例 | 10-15%  | 功能说明 |
| 中等示例 | 20-25%  | 设计意图 + 边缘情况 |
| 复杂示例 | 30-40%  | 完整逻辑流 + 决策点 |

**注释模板**：

```typescript
// ============================================
// 模块：任务上下文计算
// 目的：为 Agent 提供执行所需的代码库上下文
// ============================================

/**
 * 计算任务上下文
 * @param taskPath - 任务文件路径
 * @param options - 配置选项
 * @returns 包含文件列表和依赖图的上下文对象
 *
 * 设计决策：
 * - 使用广度优先遍历确保依赖顺序
 * - 缓存文件 AST 避免重复解析
 */
async function computeContext(
  taskPath: string,
  options: ContextOptions
): Promise<TaskContext> {
  // 1. 解析入口文件
  // 注意：使用 TypeScript Compiler API 获取完整类型信息
  const entryFile = await parseFile(taskPath);

  // 2. 构建依赖图
  // 策略：递归解析 import 语句，检测循环依赖
  const depGraph = buildDependencyGraph(entryFile);

  // 3. 排序文件（确保依赖在先）
  const sortedFiles = topologicalSort(depGraph);

  return {
    files: sortedFiles,
    dependencies: depGraph,
    metadata: extractMetadata(entryFile)
  };
}
```

### 图表使用原则

**何时使用图表**：
- ✅ 复杂流程（3+ 步骤）
- ✅ 状态变化（4+ 状态）
- ✅ 数据结构（5+ 字段）
- ❌ 简单列表（使用 Markdown 列表）

**图表位置**：
- 在文字说明之前（预览）
- 在代码示例之后（总结）

### 验证点设计

每个步骤后添加验证：

```markdown
### 步骤 2：配置项目

**操作**：创建配置文件 `superpower.config.json`

```json title="superpower.config.json"
{
  "entry": "src/index.ts",
  "output": "dist",
  "plugins": ["@superpower/typescript"]
}
```

**验证**：
```bash
npx superpower config validate
# ✅ 期望输出：Config is valid
```

**常见问题**：
- ❌ 错误：`Config file not found`
  - 解决：确保文件在项目根目录
- ❌ 错误：`Invalid JSON`
  - 解决：使用 JSON 验证工具检查语法
```
