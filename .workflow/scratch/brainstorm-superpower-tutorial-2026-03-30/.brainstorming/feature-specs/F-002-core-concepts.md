## 1. 需求摘要

本功能深入讲解 SuperPower 的三大核心概念：Command（命令）、Skill（技能）、Agent（代理），帮助用户理解系统架构和工作原理。

**MUST**:
- 讲解 Command → Skill → Agent 三大核心概念
- 每个概念使用类比降低理解难度
- 每个概念配可运行示例
- 使用渐进式深度标记（基础/进阶/精通）

**SHOULD**:
- 解释概念间的交互关系
- 提供完整的命令示例
- 展示常见使用模式

**MAY**:
- 添加历史背景说明
- 提供扩展阅读链接

**MUST NOT**:
- 单章信息过载（遵循每章 3 概念规则）
- 使用复杂的技术图表（仅 ASCII）
- 假设用户已有深入理解

## 2. 设计决策

### 2.1 概念讲解顺序决策

[UD-001 RESOLVED] 采用认知优先路径：Command → Skill → Agent

**决策理由**:
1. **用户认知顺序**: 用户先"发送命令"，看到"技能触发"，再理解"代理执行"
2. **实践验证**: 快速入门已让用户执行过命令，从 Command 开始符合已有经验
3. **递进理解**: 从用户可见（Command）到系统内部（Agent）

**技术实现路径作为备选**:

### 2.5 类比使用原则

**原则**:
1. **准确性优先**: 类比不能牺牲技术准确性
2. **单一职责**: 一个概念一个类比
3. **明确界限**: 标注类比失效的地方

**Command 类比**:
- **类比**: 餐厅点单系统
- **映射**:
  - 用户 = 顾客
  - Command = 订单
  - Skill = 菜谱
  - Agent = 厨师
- **失效点**: Agent 的并发能力超出厨师类比

**Skill 类比**:
- **类比**: 乐谱与演奏家
- **映射**:
  - Skill = 乐谱（可复用的指令序列）
  - Agent = 演奏家（执行指令的实体）
  - 参数 = 演奏风格/速度
- **失效点**: Skill 的条件触发超出乐谱类比

**Agent 类比**:
- **类比**: 专业顾问团队
- **映射**:
  - Agent = 顾问（专业知识 + 执行能力）
  - Subagent = 子顾问（分工协作）
  - 上下文 = 项目背景
- **失效点**: Agent 的零上下文继承超出团队类比
