# F-001: 快速入门教程（10分钟）

**功能 ID**: F-001
**功能名称**: 快速入门教程
**优先级**: P0（必须实现）
**预估字数**: 1800 字
**创建日期**: 2026-03-30

---

## 1. 需求摘要

本功能旨在让新用户在 **10 分钟内**完成从零到首次成功操作的完整体验，包括安装验证、环境检查和第一个命令执行。

**MUST**:
- 10 分钟内完成安装 → 验证 → 首次命令全流程
- 每个步骤都有明确的验证标准
- 提供可操作的错误恢复路径

**SHOULD**:
- 每步不超过 3 个新概念
- 使用最小化步骤策略
- 提供预期输出示例

**MAY**:
- 添加平台特定的提示
- 包含视频演示链接（可选）

**MUST NOT**:
- 解释"为什么"，只展示"怎么做"
- 包含任何未经验证的命令
- 跳过错误处理说明

---

## 2. 设计决策

### 2.1 认知负荷管理策略

[EP-001 RESOLVED] 采用渐进式深度标记，快速入门仅使用 `📊 基础层` 内容。

**原则**:
1. **最小化新概念密度**：每步最多 3 个新概念
2. **黑盒抽象**：将 Agent/Skill/Command 视为黑盒，只关注输入输出
3. **延迟深入**：所有"为什么"的解释链接到核心概念章节

**示例结构**:
```markdown
## 步骤 1: 安装 SuperPower

[执行命令]

[验证命令]

> 💡 **想深入了解？** [@link(core-concepts/installation)] 查看安装原理
```

### 2.2 技术深度控制

[跨角色分析 RESOLVED] 技术领域专家要求解释"为什么"，UX 专家要求最小认知负荷。解决方案：

**分层解释策略**:
- **快速入门**：仅展示"怎么做"
- **核心概念**：简要解释"为什么"
- **高级主题**：完整的技术原理

**具体规则**:
- ✅ 允许："执行这个命令来完成 X"
- ❌ 禁止："执行这个命令，因为它会调用 Agent 的 Y 机制..."

### 2.3 错误处理双层架构

[跨角色分析 RESOLVED] 架构师要求完整故障排除，UX 专家要求可操作的错误信息。

**第一层：简化恢复**
```markdown
### 错误：command not found

**快速修复**：
```bash
npm install -g @superpower/cli
```

[查看完整故障排除 →]
```

**第二层：完整故障排除**
```markdown
<details>
<summary>展开查看完整故障排除</summary>

### 可能原因
1. PATH 未配置...
2. 权限不足...
...
</details>
```

### 2.4 平台兼容性处理

**决策**:
- 主流程使用跨平台命令
- 平台特定问题放入"平台提示"框
- 验证命令自动检测平台

**示例**:
```markdown
**验证安装**：

```bash
# 自动检测平台
superpower --version
```

> **🖥️ Windows 用户**：如果出现权限错误，请以管理员身份运行终端
> **🐧 Linux/macOS 用户**：如果出现 EACCES 错误，请运行 `sudo npm install -g @superpower/cli`
```

### 2.5 代码示例验证标准

[UD-003 RESOLVED] 使用 L1（语法）验证级别。

**验证要求**:
- 所有命令必须通过 Markdown lint 检查
- 所有代码块必须标注语言类型
- 关键命令标注 `@verified` 标记

**示例**:
```markdown
```bash @verified

```

**预期输出**:
```

```
```

## 4. 约束与风险

### 4.1 技术约束

| 约束项 | 影响 | 缓解策略 |
|--------|------|---------|
| 平台差异 | 命令在不同平台行为不一致 | 使用跨平台工具，平台特定问题放入提示框 |
| 版本迭代 | 教程可能随 SuperPower 更新而过时 | 记录版本号，添加版本检查命令 |
| 网络依赖 | 安装需要网络连接 | 提供离线安装指南（附录） |

### 4.2 内容风险

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| 信息过载 | 10 分钟内无法完成 | 严格控制每步新概念 ≤ 3 个 |
| 缺乏实践 | 用户只看不练 | 每步后跟验证命令 |
| 首次失败 | 用户因错误而放弃 | 双层错误处理，简化恢复优先 |

### 4.3 认知风险

| 风险 | 缓解策略 |
|------|---------|
| "为什么"缺失 | 所有深入解释链接到核心概念，保持快速入门的简洁 |
| 概念模糊 | 使用类比，但避免过度简化导致误解 |
| 步骤跳跃 | 每步结束验证，确保用户在同一起点继续 |

## 附录 C: 错误处理示例

```markdown
## 故障排除

### 错误：command not found

**快速修复**（第一层）：

```bash
# 重新安装
npm install -g @superpower/cli

# 验证 PATH
echo $PATH | grep -q "npm" && echo "PATH OK" || echo "Fix PATH"
```

[展开查看完整故障排除 →]

<details>
<summary>完整故障排除（第二层）</summary>

#### 可能原因

1. **PATH 未配置**
   - 检查 npm 全局安装路径：`npm config get prefix`
   - 将路径添加到 PATH 环境变量

2. **权限不足**
   - Linux/macOS: 使用 `sudo npm install -g @superpower/cli`
   - Windows: 以管理员身份运行终端

3. **npm 版本过旧**
   - 更新 npm: `npm install -g npm@latest`

#### 验证步骤

```bash
# 检查 npm 全局路径
npm config get prefix

# 检查 PATH
which superpower  # Linux/macOS
where superpower  # Windows
```

</details>
```
