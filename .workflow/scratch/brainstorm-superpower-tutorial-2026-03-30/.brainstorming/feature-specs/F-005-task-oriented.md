# F-005: 任务导向指南

**功能 ID**: F-005
**功能名称**: 任务导向指南
**优先级**: P1（应该实现）
**预估字数**: 5000 字（每个任务类型 1000 字）
**创建日期**: 2026-03-30

---

## 1. 需求摘要

本功能提供按用户意图组织的独立任务指南，支持用户快速找到"如何完成 X"的解决方案，每个指南可独立学习。

**MUST**:
- 按用户意图分类（我想做什么？）
- 每个指南可独立学习
- 提供多维索引（意图、模式、技能）
- 每个指南包含问题诊断

**SHOULD**:
- 包含前置检查和执行步骤
- 提供常见错误处理
- 支持快速查找

**MAY**:
- 添加难度标记
- 提供时间估算

**MUST NOT**:
- 假设用户已完成所有前置教程
- 使用技术术语作为主分类

### 2.3 任务分类体系

**一级分类**（用户意图）:
1. **功能开发** - 添加新功能
2. **Bug 修复** - 修复问题
3. **代码重构** - 改进结构
4. **性能优化** - 提升性能
5. **测试添加** - 补充测试
6. **代码审查** - 审查代码

**二级分类**（技术模式）:
- TDD 模式
- 调试模式
- 集成模式
- 重构模式

**三级分类**（技能）:
- Brainstorm
- Spec
- Implementer
- Reviewer

### 2.4 可发现性设计

**搜索优化**:
- 使用用户语言（非技术术语）
- 提供同义词映射
- 标注常见场景

**示例**:
```markdown
## 功能开发

**你可能想**:
- 添加用户登录
- 实现 API 端点
- 添加数据验证
- 创建新页面

**相关技能**: Brainstorm, Spec, Implementer
```

### 2.5 问题诊断组件

**目的**: 帮助用户确认这是正确的任务

**模板**:
```markdown
## 问题诊断

**这个任务适合你，如果**:
- ✅ 你要添加全新的功能
- ✅ 功能需求已经明确
- ✅ 你有权限修改相关代码

**考虑其他任务，如果**:
- ❌ 你要修复现有问题 → [Bug 修复](bug-fixing.md)
- ❌ 你要改进代码结构 → [代码重构](refactoring.md)
```

## 4. 约束与风险

### 4.1 技术约束

| 约束项 | 影响 | 缓解策略 |
|--------|------|---------|
| 任务重叠 | 用户不知道选哪个 | 问题诊断组件 |
| 前置依赖 | 指南可能假设过多知识 | 明确前置检查 |

### 4.2 内容风险

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| 分类不清 | 用户找不到任务 | 多维索引 + 搜索优化 |
| 任务过多 | 用户迷失 | 提供推荐路径 |
| 缺乏实践 | 只有理论 | 每个指南可执行 |

### 4.3 UX 风险

| 风险 | 缓解策略 |
|------|---------|
| 导航混乱 | 清晰的问题诊断 |
| 错误任务 | 交叉引用其他任务 |
| 认知过载 | 每个指南独立完整 |
