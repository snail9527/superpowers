## 二、澄清解决记录

### 2.1 F-002: 概念讲解顺序

**原冲突**:
- 技术领域专家: Agent → Skill → Command（技术实现顺序）
- UX 专家: Command → Skill → Agent（用户认知顺序）

**解决方案**:
- ✅ 主路径: 认知优先（Command → Skill → Agent）
- ✅ 备选路径: 技术实现（侧边栏提供）
- ✅ 依赖图导航: 支持用户自主选择

**应用位置**:
- F-002: 核心概念讲解（主要应用）
- F-006: 角色导向学习路径（路径编排）

---

### 2.2 F-004: TDD 讲解深度

**原冲突**:
- 技术领域专家: 必须深入"为什么 TDD"
- UX 专家: 优先展示"怎么做"

**解决方案**: 分层说服策略

| 层级 | "为什么"深度 | 位置 |
|------|------------|------|
| 快速入门 | 仅展示"怎么做" | F-001 |
| 核心概念 | 简要解释"为什么" | F-002 |
| **质量保证** | **完整"常见合理化反驳"** | **F-004** |

**应用位置**:
- F-004: 包含 12 个常见合理化反驳
- 每个"为什么"都有数据支撑

---

### 2.3 F-005: 索引组织方式

**原冲突**:
- UX 专家: 按用户意图
- 技术领域专家: 按技术模式
- 系统架构师: 按 SuperPower 能力

**解决方案**: 多维索引

**主索引**: 用户意图（UX 优先）
```markdown
- 添加新功能
- 修复 Bug
- 重构代码
```

**次级索引**: 技术模式（技术专家）
```markdown
- TDD 模式
- 调试模式
- 集成模式
```

**侧边栏**: 相关技能（架构师）
```markdown
- Brainstorm
- Spec
- Implementer
```

**应用位置**:
- F-005: 任务导向指南（索引页）
- 每个任务指南元数据
