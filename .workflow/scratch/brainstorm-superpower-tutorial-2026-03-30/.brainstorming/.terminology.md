# SuperPower 核心术语表

| 术语 (英文) | 术语 (中文) | 定义 | 别名 | 类别 |
|-----------|-----------|------|------|------|
| **Agent** | 代理/智能体 | AI 编程助手（如 Claude Code、Cursor、Gemini CLI），能够理解指令、执行任务、使用工具的 AI 实体 | Coding Agent, AI Assistant | 核心 |
| **Skill** | 技能 | 可组合的模块化工作单元，定义了 Agent 如何处理特定类型任务。包含触发条件、执行流程和验收标准 | Workflow, Task Pattern | 核心 |
| **Command** | 命令 | 用户向 Agent 发出的指令请求，触发相应的 Skill 执行 | User Request, Prompt | 核心 |
| **Subagent** | 子代理 | 为执行特定独立任务而分派的新 Agent 实例，具有隔离的上下文和明确的任务范围 | Worker Agent, Task Agent | 技术 |
| **Workflow** | 工作流 | 由多个 Skill 组成的端到端流程，定义了从开始到完成的完整任务执行序列 | Pipeline, Process | 核心 |
| **TDD** | 测试驱动开发 | 先编写失败的测试，再编写最小代码使测试通过，最后重构的开发方法论 | Test-Driven Development | 技术 |
| **Spec Compliance** | 规范遵循 | 确保实现严格符合设计规范的质量检查流程 | Specification Check | 技术 |
| **MCP** | 模型上下文协议 | 标准化的 AI 模型与工具/数据源之间的通信协议 | Model Context Protocol | 技术 |
| **Git Worktree** | Git 工作树 | Git 的并行分支管理机制，允许在同一仓库中同时检出多个分支到不同目录 | Parallel Branch | 技术 |
| **Code Review** | 代码审查 | 对代码质量、规范遵循性和正确性的系统化检查过程 | Quality Review | 技术 |

---

## 术语关系图

```
User (Command)
    ↓
Agent (主代理)
    ↓
Skill (触发条件匹配)
    ↓
Workflow (执行流程)
    ↓
Subagent (任务分派，可选)
    ↓
TDD + Code Review (质量保证)
```

## 核心概念层次

1. **用户层**: Command（用户指令）
2. **交互层**: Agent（AI 助手）
3. **逻辑层**: Skill（技能模块）
4. **执行层**: Subagent（子代理，可选）
5. **质量层**: TDD + Code Review（质量保证）
