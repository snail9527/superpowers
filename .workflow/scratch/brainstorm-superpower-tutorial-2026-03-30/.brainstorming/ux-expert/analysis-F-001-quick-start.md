## 1. 功能概述

**目标**：在 10 分钟内让用户完成首次成功操作，建立对 SuperPower 的初步信任和兴趣。

**核心价值主张**：
- 快速验证工具价值
- 低门槛上手
- 即时成功体验

---

## 2. 用户目标分析

### 2.1 主要用户目标

**一级目标（显性）**：
- "我想快速看看这个工具能做什么"
- "我想验证安装是否成功"
- "我想获得第一个成功体验"

**二级目标（隐性）**：
- "我想判断这个工具是否值得投入时间学习"
- "我想了解这个工具的基本能力"
- "我想避免在配置上浪费太多时间"

### 2.2 用户心理状态

**初始状态**：
- 好奇但谨慎
- 担心配置复杂
- 希望快速得到反馈

**目标状态**：
- 获得首次成功体验
- 建立对工具的信任
- 激发继续学习的兴趣

**转化路径**：
```
好奇（阅读快速入门）
  ↓
尝试（按照步骤操作）
  ↓
成功（看到命令执行成功）
  ↓
信任（相信工具能解决问题）
  ↓
投入（愿意继续学习核心概念）
```

### 2.3 成功的心理强化

**首次成功的重要性**：
- 建立"我也能做到"的自我效能感
- 消除对技术复杂度的恐惧
- 创造积极的情感联结

**强化机制**：
```
操作 → 即时反馈 → 明确成功 → 正向情绪 → 继续动机
```

## 3. 10 分钟快速体验设计

### 3.1 时间分配策略

**总时间预算**：10 分钟

**推荐分配**：
```
安装验证（2分钟）
├─ 检查系统要求（30秒）
├─ 运行安装命令（30秒）
└─ 验证安装成功（1分钟）

第一个命令（5分钟）
├─ 理解命令结构（1分钟）
├─ 执行命令（1分钟）
├─ 观察输出（2分钟）
└─ 理解结果（1分钟）

结果验证（3分钟）
├─ 查看生成文件（1分钟）
├─ 理解工作空间结构（1分钟）
└─ 确认操作成功（1分钟）
```

### 3.2 步骤简化原则

**最小化认知负荷**：
- 每步只做一件事
- 每个命令都提供解释
- 每个输出都标注预期结果

**避免早先进阶**：
- ❌ 不解释架构原理
- ❌ 不介绍配置选项
- ❌ 不讨论错误处理
- ✅ 专注于"能跑起来"

### 3.3 示例流程设计

**步骤 1：安装验证（2 分钟）**
```bash
# 检查安装
$ superpower --version

# 预期输出
SuperPower v5.0.0
✓ 安装成功！

# 如果失败
❌ 显示错误信息
💡 查看故障排除部分
```

**步骤 2：第一个命令（5 分钟）**
```bash
# 执行简单的 brainstorm 命令
$ superpower brainstorm "添加用户登录功能"

# 预期输出
🔄 启动 brainstorming session...
✓ Session created
📁 Workspace: .workflow/scratch/brainstorm-add-login-2026-03-30/
💡 Ideas generated: 5 concepts
```

**步骤 3：查看结果（3 分钟）**
```bash
# 查看生成的文件
$ ls .workflow/scratch/brainstorm-add-login-2026-03-30/

# 预期输出
ideas.md  session.json  workspace.json

# 查看生成的内容
$ cat .workflow/scratch/brainstorm-add-login-2026-03-30/ideas.md

# 预期输出：包含 5 个不同的登录功能实现想法
```

## 5. 用户体验风险点

### 5.1 安装失败风险

**风险描述**：用户在安装阶段遇到错误，可能直接放弃。

**缓解策略**：
1. **明确错误信息**：
```
❌ 错误：Node.js 版本过低

当前版本：v14.0.0
要求版本：≥ v18.0.0

[解决方法]
1. 下载最新版 Node.js: https://nodejs.org/
2. 安装后重新运行: npm install -g superpower
3. 验证: superpower --version
```

2. **提供故障排除清单**：
- 常见错误及解决方案
- 环境检查命令
- 获取帮助的渠道

### 5.2 时间超出预期风险

**风险描述**：用户实际花费时间超过 10 分钟，产生挫败感。

**缓解策略**：
1. **分段里程碑**：
```
📍 里程碑 1/3：安装验证（预计 2 分钟）
✓ 完成！

📍 里程碑 2/3：第一个命令（预计 5 分钟）
→ 进行中...

📍 里程碑 3/3：结果验证（预计 3 分钟）
○ 待完成
```

2. **时间弹性提示**：
```
⏱️ 时间提示

如果遇到问题，不要担心！
- 安装验证：通常 1-2 分钟
- 第一个命令：通常 3-5 分钟
- 结果验证：通常 2-3 分钟

如果某一步耗时超过预期的 2 倍，建议查看故障排除部分。
```

### 5.3 结果理解困难风险

**风险描述**：用户看到输出但不理解其含义。

**缓解策略**：
1. **输出注释化**：
```bash
$ superpower brainstorm "添加用户登录功能"

🔄 启动 brainstorming session...  # Agent 正在处理你的请求
✓ Session created                 # 成功创建了 brainstorming session
📁 Workspace: ...                 # 结果存储在这里
💡 Ideas generated: 5 concepts    # 生成了 5 个不同的想法
```

2. **结果解读指南**：
```
📊 结果解读

你刚刚做了什么：
1. 向 SuperPower 发送了一个 brainstorming 命令
2. Agent 分析了你的需求（"添加用户登录功能"）
3. Skill 生成了多个实现方案（5 个概念）
4. 结果保存在工作空间目录中

这意味着：
- SuperPower 能理解自然语言需求
- Agent 会生成多个方案供你选择
- 所有结果都保存在工作空间中

下一步：
→ 查看 ideas.md 了解具体方案
→ 选择最佳方案继续开发
```

**友好的错误信息**：
```bash
$ superpower brainstorm "我的需求"

❌ 错误：工作空间目录已存在

目录：.workflow/scratch/brainstorm-我的需求-2026-03-30/

[解决方案]
1. 使用 --force 覆盖现有会话
   $ superpower brainstorm "我的需求" --force

2. 使用 --resume 恢复现有会话
   $ superpower brainstorm --resume brainstorm-我的需求-2026-03-30

3. 使用新名称创建新会话
   $ superpower brainstorm "我的需求 v2"

[相关文档]
→ 故障排除指南
→ 命令参数参考
```

## 8. 设计决策记录

### 8.1 10 分钟时间预算

**决策**：设定 10 分钟作为快速入门的时间预算。

**理由**：
- 符合现代用户的注意力跨度
- 足够完成基本的安装和验证
- 不会因为时间过长而失去耐心

**风险**：用户可能在 10 分钟内无法完成。

**缓解**：
- 提供分段里程碑
- 强调"可以分多次完成"
- 提供故障排除快速通道

### 8.2 简化 vs 完整性

**决策**：在快速入门中优先简化而非完整性。

**理由**：
- 首次体验的成功感比知识完整性更重要
- 过多细节会增加认知负荷
- 后续章节可以补充详细内容

**风险**：用户可能形成不完整的概念理解。

**缓解**：
- 明确标注"简化版本"
- 在每步提供"深入了解"链接
- 在检查点澄清核心概念
